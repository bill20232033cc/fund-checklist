请基于上述风险相关数据，写一段「核心风险与风险分级」分析。

【数据验证规则 - 必须严格遵守】
1. 引用数据表中的数字时，必须使用原始精度（如 "10,095,099,672.67"），不得缩写（如 "100.95亿"）
2. 如需使用亿元等单位，必须在括号中注明原始数字
3. 如果某年数据缺失，必须明确声明"该年数据缺失"，不得跳过或用其他年份数据替代
4. 分析前先逐条核对数据表中的关键数字，确认趋势方向（增长/下降/波动）

【风险分析框架 - 逐步回答】
请按以下顺序逐步分析，每步都必须引用数据表中的具体数字：

第一步：识别核心风险
- 从数据表中找出最关键的 1-2 个风险
- 区分结构性风险（长期存在）和阶段性风险（短期波动）
- 引用具体数据说明为什么这是核心风险

第二步：风险严重程度分级
- 基于数据表中的具体数字，将核心风险分为三级：
  - 高风险：数据缺失/异常导致无法评估（需声明具体缺失项）
  - 中风险：有数据但趋势恶化（引用数据）
  - 低风险：数据正常、无异常信号
- 每级必须引用数据表中的具体数字作为依据
- 禁止使用"一票否决""致命""不可接受"等绝对化表述

第三步：信息缺口分析（必须回答）
- 必须以「当前最大的信息缺口是 [具体字段/数据]」开头，明确写出
- 这个缺口如果被填补（获取到正确数据），会如何改变你的风险判断？
- 如果数据确认是错误的，结论会如何变化？
- 如果数据确认是正确的，结论会如何变化？

【投资建议边界 - 必须严格遵守】
禁止：
- "建议买入/卖出/持有/替换"
- "不具备可跟踪的基础"（这是变相的"不建议持有"）
- "应该/不应该投资"
- 任何直接或间接的操作建议

允许：
- "建议关注 [具体风险点]"
- "需持续跟踪 [具体变量]"
- "在 [条件] 得到确认前，风险特征难以评估"
- 中性的风险描述和跟踪建议

【语言规范 - 必须严格遵守】
禁止使用以下情绪化/绝对化表述：
- "极不寻常""严重异常""致命""大涨大跌""剧烈波动"
- "一票否决""不可接受""无法容忍"
- "投资者应""投资者需要""建议投资者"

替代方案：
- "极不寻常" → "偏离历史均值" 或 "数据异常"
- "大涨大跌" → "波动幅度较大" 或 "年度间差异显著"
- "致命" → "可能导致判断改变" 或 "构成关键不确定性"
- "一票否决" → "高风险" + 具体依据
- "投资者应" → 删除，改为中性描述

【Ch6 正例 vs 反例】
✅ 正例（风险严重程度分级）：
「据数据表，2024-2025年前五大持仓集中度为0.00%，偏离2021-2023年20%-22%的历史均值。该异常属于高风险：持仓数据缺失导致无法评估基金近两年的集中度变化，构成关键信息缺口。若数据确认为未披露，需重新评估基金的信息透明度；若数据确认为采集错误，风险等级可下调。」

❌ 反例（投资建议——Ch6 禁止）：
「该风险构成一票否决（注：使用了禁止的绝对化表述）」

❌ 反例（情绪化表述——Ch6 禁止）：
「这一数据异常极不寻常，属于严重的信息披露异常，直接导致致命的风险判断困难。」

✅ 正例（信息缺口回答）：
「当前最大的信息缺口是2024-2025年前五大持仓集中度数据。若该数据确认为未披露（而非采集错误），风险等级将从高上调至极高，因主动基金的持仓透明度是评估其策略执行能力的基础。」
