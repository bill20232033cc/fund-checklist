## 重点核查

- Service/CLI 边界：`FundReadingService.read_local_report()` 在装配 ToolService 后只以 `document_id` 与 `query` 调用 Host；CLI 测试覆盖 plain text 输出、exit code 与本地路径/Docling JSON/local_import_id redaction。
- Repository 复用：Service 先导入本地 PDF 得到内容身份，再尝试 `repository.load_store(document_id)`；completed catalog 存在时不重复 conversion，catalog record 指向的 Docling JSON 缺失时 fail-closed，不自动 repair/reconvert。
- 失败分类：Service 透传 `DocumentToolError` 稳定 code；CLI classified failure 仍返回 exit code 2，unexpected failure 返回 exit code 1。
- Table-backed search：`DoclingDocumentStore.search()` 同时聚合 section 与 table 候选；table 候选返回 `table_ref`、table locator 与 citation。
- Bounded table rows：table 搜索投影只包含 caption 与 `DEFAULT_TABLE_MAX_ROWS` 内的行；测试覆盖越界行不命中。
- 禁止项：未看到 Agent policy、LLM adapter、CLI 参数、routing、计算框架、报告方向、raw Docling JSON/path/cache/local_import_id 泄漏的 diff。
